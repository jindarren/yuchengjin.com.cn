Targeted advertising reaches users based on various traits, such as demographics or behaviour. However, users are often reluctant to accept ads. We hypothesise that users are more open to targeted advertising if they can inspect, control and thereby understand the process of ad selection.

The Facebook Ads provides text based explanation and configuration. The Facebook Ads page explains why a user is seeing an ad and allows to configuring ad preferences.

How about visualizing the process of ad selection?

Flow charts have been found effective for presenting complex structural relationships of cause and effect, being superior to text-only explanations especially for inexperienced users. To investigate, we designed a visualization "AdFlow" in a Facebook web app.

DESIGN PROCESS

Our research work follows the user-centered design approach. We will present how we reach to our final design "AdFlow" to show transparency and user control for personalized ads. In the end, we evaluated how the visualization affects the user perceived quality of ads.

The selection of ads is based on the predicted user profile and the recorded activities on social media like Facebook. However, currently users have poor understanding of ads selection and very limited control over the ads. Therefore, we aim at designing a visualization where people can inspect and control the logic of ad selection.

Different prototypes design

Version A

The prototype of version A consists several charts which enables subjects to inspect and change the generated user model from their Facebook data and gain insight into the relationship between the user model and the number and categories of available ads. By showing such visualizations, we expect subjects will understand why a particular ad is shown to them. By adjusting the user model, they can control the ad selection process.

Pros

  • The user model is shown clearly.
  • Selection of ad categories is useful.

Cons

  • The process of ad selection is not explained yet.
  • The number of available ads in the visualizations is not useful.
  • The visualizations looks complex.
  • Difficult to understand the values of predicted user personality.
Version 2

The visualizations of Version 2 mainly use textual information for showing user model and ad selection. Figure 3 a) and b) show the user model and the explanation why the current ad is selected. Figure c) is control panel for changing the user model.

Pros

  • Easy for the novice to use.
  • The information is well organized to show.

Cons

  • Textual explanation for ad selection is insufficient.
  • Difficult to judge the personality value.
Final version

The final version comes out after evaluating the previous two prototypes. The prototype mainly consists of three parts, a): the shown ad and a pair of rating buttons; b): visualization AdFlow showing the user model and the process of ad selection; c): the control panel for configuring the predicted user model.

USER STUDY

In order to investigate the meaningfulness of transparency and user control on ads, we evaluated AdFlow for about 70 online ads with 200 participants in a between-subjects study. In specific, we measure the success of ads based on four key aspects:

  • Perceived Quality of the ad (interest match, context match, attractiveness and annoyance)
  • Behavioral Intention of the user (willingness to click,purchase and see)
  • Understanding of the ad selection
  • User Attitude towards the ad (satisfaction, acceptance confidence and trust)

Results

The following table shows the qualitative feedback of 200 participants, then we also performed statistical tests for the feedback data to check the statistically significance for each of these investigated aspects.

The results of user study can be concluded as follows,

  • Transparency can improve a user's behavioral intention.
  • User control can increase a user's understanding of the ad selection process.
  • The most successful approach is to show transparency and user control together. This way, we could simultaneously improve aspects of Quality, Behavioral Intention, and Understanding.
  • User Attitude towards advertising could not be improved by transparency and user control.

Visual search aims to find information based on visual features of objects, and it is extremely useful when specific keywords for describing an object are unknown. Visual search has been considered as one of the most exciting trends in e-commerce.

What do we benefit from visual search technology?

It shows the possibility to select or point to information by the user (e.g. on touch screen) whereby the content is automatically recognized and the user is automatically directed to a Web page with product information on the selected content.

A number of mobile apps of visual search such as Slyce, Amazon Flow, Snap Fashion have been developed, in which users can find products by taking a photo. However, taking a photo to precisely select a specific product in a video is difficult.

CircleBuy is designed especially for searching products in videos with seamlessly integrating visual search while watching videos. We actually find a second screen application named TheTake which allows users to discover products from their favourite videos.

User Study

We employed a mix of quantitative and qualitative methods to evaluate if our approach can be meaningful towards facilitating the process of purchasing products that appear in videos. More specifically, we investigated if:

  1. CircleBuy facilitates the entire process of finding and buying products that appear in videos.
  2. The similarity of resulting products of visual search influences users’satisfaction and therefore user adoption.
  3. Showing a 3D model view adds value for making purchase decision(s).
  4. Sharing a product on TV adds value for making purchase decision(s).

Setting up

We set up our evaluation in an office with a 60 inch TV and a sofa (see Figure 5) to mimic a living room setting. The whole procedure of evaluation was recorded by a Logitech camera.

Method

We use record the users activities on app in a log file. Apart from using questionnaires, during the evaluation we also employed the think-aloud protocol to collect subjects' feedback.

Results

  • Our app can facilitate the entire process of finding and buying products that appear in videos.
  • The similarity of resulting products of visual search tends to influence users’ satisfaction and therefore user adoption.
  • It is more likely that 3D model view and sharing feature add limited value for making purchase decision(s).
  • Around 80% of subjects thought the AR view can increase the meaningfulness of 3D models.

Warehouse order picking

Process of retrieving items from storage locations in response to specific customer request.

  • Most laborious in a typical warehouse
  • Up to 55% of warehouse operating cost
  • Direct influence on order accuracy and delivery time

Design Goal

This work aims to explore appropriate adaptive rules for order picking. The system should allow users to interact in a multi-modal way (vocal and visual interaction). By virtue of wearable devices and ambient intelligent environment, the user experience of picking task has been enhanced apparently with designed adaptive features.

DESIGN

Scenario

My colleagues firstly interviewed several workers who work in a local warehouse with intelligent logistics system. After analyzing the feedback collected from interview we outline such a scenario.

Figure 2 shows logistics scenario: (1) RFID based Production with automatic material and production flow steering, (2) RFID Temperature Tracking with Supply Chain temperature Sensor, (3) Warehouse Management with an RFID-Forklift, (4) Intelligent Picking, (5) RFID Labelling trade units, (6) Pack and Ship & Good Receipt with RFID goods issue and palette aggregation with RFID hand reader.

Adaptive rules

After interviewing the warehouse workers with our proposed adaptive rules, we picked the most five popular rules for the prototypes.

Fragile object - The rationale of this rule is that when the worker is about to pick a fragile object, the multi-modal UI should switch to only-vocal modality in order not to distract the user while picking the item.

  • Event: the right shelf has been reached
  • Condition: the worker has to pick a fragile item and the current modality is not only-vocal
  • Action: switch from multi-modal to only-vocal modality

Picking timeout - The user has just reached the destination shelf of the item but there is no confirmation of the actual item picking. The application then assumes that the worker is distracted/confused and/or not able to recognize the item to pick, then it provides again info on the item, both graphically and vocally.

  • Event: the user has reached the destination shelf
  • Condition: there has not been confirmation of the item picking and the user interface is multi-modal
  • Action: the application visualizes an image representing the item to pick, and simultaneously repeats the item name vocally

Order visualization for experienced workers - If the user has good knowledge of the warehouse shelf organisation, there is no need to show associated path information: the application adapts accordingly.

  • Event: beginning of a session with the HMD
  • Condition: the user is a warehouse expert
  • Action: the application hides the information about how to reach the different shelves

Traffic Jam - There are multiple workers who are expected to approach the same path at the same time: the application adapts in order to minimise the risk of workers to wait for other people before picking the items.

  • Event: order completed
  • Condition: multiple pickers are expected to approach the same path at the same time and the path optimization preference is selected
  • Action: the application shows the blocked path suggesting a different route

Noisy environment - The environment gets noisy, then the multi-modal application switch to ‘only-graphical’ modality.

  • Event: the environment gets noisy
  • Condition: the application is using both the graphical and vocal modality for interacting with the user
  • Action: the application switches to the only-graphical modality

UI Design

EVALUATION

In order to reveal those adaptation rules which are generally perceived as the best and most reasonable ones, we conducted two rounds of user study for low-fidelity and high fidelity prototypes. Specifically, we assessed the user perceived quality of the adaptation rule in a 9-item questionnaire.

  • User's awareness
  • Appropriateness
  • Comprehensibility
  • Effectiveness of performance
  • Effectiveness of user experience
  • Error-prevention
  • Continuity
  • Intuitiveness
  • General likeability

Collaborative design platform (CDP) focuses on making interaction between analogue urban massing models and interactive simulations direct and intuitive. Technically the major part of CDP is a custom-built multi-touch table. Architects are able to work and design in a familiar,reliable and novel way. In addition, the real-time interaction allows the architects to get the designing effects immediately. E.g., the real-time simulation of building shadow is available after setting a time period.

Multi-touch table is flexible and can be customized. It supports various interaction methods. According to the design scenarios, we designed three different interaction methods (traditional touch, tangible interaction, and finger gesture interaction) on multi-touch table for the simulation program of building shadow.

This work aims at finding the most suitable interaction method for CDP environment. All above mentioned interaction methods will be developed and then evaluated by students of architecture department.

PRELIMINARY DESIGN

In Figure 2, we can see the building shadow is affected by solar elevation angle and terminator angel. Because solar elevation angle depends on hour angle in local solar time and current sun declination, the user can control shadow via adjusting the period of time. The time units of our system include hour, date and month. Since we have chosen location the local latitude is not considered. Therefore we need design a user interface for changing hour, date and month for simulating building shadow.

Design Tasks

  • Design a user interface for setting a period of time for shadow simulation.
  • Implement three different interaction methods to set time.

DESIGN

After presenting the paper prototype to our client, we received new requirements:

  • The design of GUI should be consistent for all three interaction methods.
  • Visually the GUI should not be too complex.
  • The gesture should be easy to learn and remember.

New design

Traditional touch

Figure 4 shows new traditional touch user interface. The spinner is used to set specific time unit value. The rectangle button in the center of circle is used to switch time units. The green point represents start time and the red one is end time.

Tangible interaction

New tangible interaction has no visible interactive elements on touch surface. We designed fiducial marks to set time. Two marks are the triggers of setting start time and end time. The rest three marks are used for setting different time units. The time unit can be set by changing the mark orientation.

Figure 6 shows a cubic and each of its facade has a fiducial mark, and the edges of a facade are labeled by the name of fiducial marks on its adjacent facades. This design helps the user find the correct fiducial mark quickly.
